mel

A tiny CLI that makes common Git workflows feel friendly.

Install

curl -L -o /usr/local/bin/mel https://raw.githubusercontent.com/davefowler/melcurial/main/mel
chmod +x /usr/local/bin/mel

Commands

Use these commands in your project folder. Plain language, safe defaults, and helpful prompts.

mel start           # creates your workspace branch (prompts for a name if omitted)
# edit files...
mel sync            # saves/stashes your changes, updates with main, and pushes
mel status          # shows what's going on
mel publish         # runs checks and ships your changes to main

Tips:

  • If mel asks what to do with local changes, choose Save or Stash.
  • If mel asks to confirm adding files, review the list and type y to continue.
  • If mel offers to open a pull request, say yes and follow the link.

Engineer‑oriented descriptions and full list:

mel start [name]    # create/reset your workspace branch from latest main (defaults to your name)
mel save            # commit-all with confirmation → rebase/merge onto main per config → push
mel sync            # save/stash prompt → update branch with latest main (rebase by default) → push
mel update          # same as sync (alias)
mel status          # prints ahead/behind, dirty files, last commit, plus `git status`
mel publish         # run tests/hooks → FF merge branch into main → push main → rebase branch → push

Advanced:

mel diff            # show staged and unstaged diff stats
mel open repo       # open remote repo page
mel open branch     # open current branch page
mel open pr         # open compare PR URL (GitHub)
mel reset           # hard reset your workspace branch to latest main and force‑push
mel run <name>       # run configured script; supports `--` to pass extra args

See Config for scripts, hooks, update strategy, and non‑interactive mode.
