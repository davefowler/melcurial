mel

A tiny CLI that makes a few Git workflows feel friendly.

start save deploy freshstart status pull test

Install

curl -L -o /usr/local/bin/mel https://raw.githubusercontent.com/your/repo/main/mel
chmod +x /usr/local/bin/mel

Quick start

mel start           # prompts for a name if omitted
# edit files...
mel sync            # save/stash, update with main, push
mel status          # see what's going on
mel publish         # run tests, fast-forward merge into main

Configure tests (two ways)

mel test runs your project's tests using one of the following setups:

1) Define a scripts.test command

Create .mel/config.json and add a test entry under scripts:

{
  "scripts": {
    "test": "pytest -q --disable-warnings"
  }
}
mel test

2) Use your package manager's test (npm/yarn/pnpm)

If your repo already has a package test (e.g. npm run test), mel will use it by default. You can also set it explicitly:

{
  "allow_package_scripts": true
}

mel will run npm run test, yarn test, or pnpm test based on your lockfile.

mel test

Bonus: package fallback also lets you run other package scripts via mel run:

mel run build
mel run lint
mel run typecheck
